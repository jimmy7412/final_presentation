Predicting Landslide Casualties from Historical News Data

By James Pierce

Problem

  • Landslides are a significant problem throughout the world.
  • According to the U.S. Geologic Survey roughly 25 - 50 people die in the U.S. every year.
  • According to the World Health Organization 18,000 people died between 1998 - 2017 from landslides.
  • This model could be used for quick assessments of world wide landslides, to determine immediate effects and damage estimation.

Metrics

  • Using Total Casualties as a final cost metric, property damage being more difficult to measure world wide.
  • I take the description text of news reports of the incidences, and attempt the find correllation of the text with casualties.
  • The model will be considered successful if the R squared score is higher than the null set.

Approach and Process

  • In order to process the text, Natural Language Processing was used.
  • However, doing that without tuning produced roughly 500,000 features to process
  • That many features made fitting models a long process, and greatly slowed down work.
  • Multiple processes were run in parallel to try and tune the models in a timely manner.
  • Some of the tuning procedures used included limiting the number of features, and

Attempted Approaches

  • Null Set: 76.4%, if all results are predicted to be zero.
  • K Nearest Neighbor Regressor plots all of the data into a graph then produces a result by finding the labeled points near the test point.
  • Multinomial Naive Bayes uses Bayes Theorem to calculate the predicted value.
  • Neural Network Multilayer Perceptron Regressor performs a series of logistic regression and weighted logistic regression to arrive at a final result.
  • Random Forest uses a series of decision trees to arrive at a result.

Model and Solution

  • None of the attempted models produced an accuaracy that was significantly greater than the Null Set
  • The Multi Layer Perceptron and Random Forest Regressor actually produced a near zero result.
  • The model that performed the best was the K Nearest Neighbor which came in at right around the Null Set result, the Multinomial Naive Bayes Model came in second, but still slightly lower.
  • The introduction of DBSCAN labels significantly slowed progress and produced no appreciable value to the set. Most of the values were outliers with the default epsilon value of .5, incleasing the value resulted in fewer outliers but increased computation time.

Proformance Evaluation

  • The models did not perform up to overall expectations.
  • With the dataset used a usable model was not able to be created.
  • Although the \(R^2\) score for the K Nearest Neighbors Regressor was somewhat high it did not perform to a level above base line.
  • Continued evaluation was slowed due to the size of the dataset.
  • One Attempt on Azure was halted due to needing 8GB of space to run clustering, and 12GB to run K Nearest Neighbors.

Impact of Findings

  • The impact of this model would be effective in immediate classification of news reports.
  • It could be used to provide alerts and begin preparation of resources and begin evacuations.

Recommendations and Next Steps

Dealing with size

  • The size of the features became a problem after adding Natural Language Processing, Having ~500,000 features created a significantly slower process. This also used resources on the computer it was run on, causing it to get hot and make the fan run constantly.
  • A potential path forward would be to offload this on to a cloud environment. By Creating a pipeline data could be fed in by an API, and results could be sent to a remote location.
